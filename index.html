Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Its recommended to use semantic elements in html when possible
  • Semantic element can indicate the role of their content and give structure to content

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Styles can be added inline, internally and externally
  • External style sheets have to be linked to the html file via the head
  • A class attribute is used to assign CCS rules to any element we choose.
  • Color can be added sematically eg. blue / green or with hexadecimal codes or hex #
  • One can style HTML elements using the elements as selectors or by using the class attribute as a selector
  • CSS allows us to style the page

GIT

  • git status: checks which branch we are currently on.
  • git checkout main: takes you back to the main branch.
  • git checkout -b branch-name: creates a new branch and switch to it
  • git add -A: adds changed files to the staging area/commit all files in the working branch.
  • git commit -m "message": Commits staged changes to the repository
  • the -m flag allows us to associate a message with our commit.
  • NOTE: every command here starts with git
